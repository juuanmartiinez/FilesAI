FileAI

Aplicación de escritorio que organiza, analiza y optimiza archivos usando modelos de lenguaje locales. Empieza por ~/Downloads y escala a más carpetas cuando esté lista.

¿Qué vas a construir?

Objetivo principal

  • MVP centrado en Downloads: escanear, analizar y organizar la carpeta de descargas.
  • Indexar el contenido de documentos para búsquedas semánticas.
  • Permitir consultas en lenguaje natural: "encuentra ese PDF sobre Python que bajé".
  • Buscar archivos por nombre o parte del nombre.
  • Mostrar estadísticas de espacio, duplicados y uso.
  • Sugerir acciones para organizar y liberar espacio.
  • Ejecutar acciones bajo supervisión del usuario.
  • Futuro: extender a otras carpetas como Documentos, Escritorio, etc.

Por qué este proyecto

  • Privacidad total: modelos locales, datos en tu máquina.
  • Aprendizaje real: tocarás NLP, RAG, bases vectoriales, sistemas de archivos y más.
  • Utilidad real: todo el mundo acumula archivos y pierde tiempo buscando.
  • Escalable: empiezas con línea de comandos y terminas con interfaz gráfica.

Arquitectura por capas

Capa 1: Exploración del sistema de archivos

Recorre carpetas, lee metadatos y detecta cambios.

  • MVP: escanear ~/Downloads.
  • Librerías: pathlib, os, hashlib.
  • Extrae: ruta, tamaño, fecha de modificación, extensión, tipo MIME.
  • Lista negra de carpetas sensibles: .ssh, .gnupg, .kimi-code, etc.
  • Escaneo progresivo: empieza con una carpeta y añade más después.
  • Futuro: detección de cambios con watchdog.

Capa 2: Extracción de contenido

Abre archivos y extrae texto útil.

  • PDFs: pymupdf.
  • Word: python-docx.
  • Excel: openpyxl.
  • Texto plano: lectura directa con detección de encoding.
  • Futuro: OCR para imágenes, transcripción de audio con Whisper.

Capa 3: Embeddings e indexación

Convierte archivos en vectores buscables.

  • Embeddings: nomic-embed-text vía Ollama.
  • Base vectorial: ChromaDB.
  • Metadatos: SQLite.

Capa 4: Modelo de lenguaje local

Entiende preguntas, genera planes, resume y clasifica.

  • Motor: Ollama.
  • Modelos recomendados: llama3.2, qwen2.5, mistral.
  • Conexión: API HTTP en http://localhost:11434.

Capa 5: Motor de acciones

Transforma las decisiones de la IA en operaciones concretas.

  • Organizar, mover, renombrar, comprimir.
  • Siempre con confirmación previa del usuario.
  • Papelera segura: mover a cuarentena en lugar de eliminar.

Capa 6: Interfaz de usuario

Lo que el usuario ve y toca.

  • Inicial: línea de comandos con Typer.
  • Avanzada: Flet para interfaz gráfica con Python.

Flujo de datos

1. Usuario selecciona carpetas a escanear (MVP: ~/Downloads)
↓
2. FileAI recorre archivos y extrae metadatos
↓
3. FileAI extrae texto de archivos soportados
↓
4. FileAI genera embeddings y los guarda en ChromaDB
↓
5. FileAI guarda metadatos en SQLite
↓
6. Usuario hace una pregunta o pide una acción
↓
7. FileAI busca en el índice (semántica + filtros)
↓
8. FileAI envía contexto + pregunta al LM local
↓
9. LM responde o propone una acción
↓
10. Usuario aprueba y FileAI ejecuta

Búsqueda por nombre vs búsqueda semántica

Búsqueda por nombre

Para cuando recuerdas más o menos cómo se llamaba el archivo.

  • No necesita modelo de lenguaje.
  • Se hace con SQL o con pathlib / find.
  • Muy rápida y precisa.
# Ejemplo con Python
from pathlib import Path

downloads = Path.home() / "Downloads"
resultados = list(downloads.rglob("*python*.pdf"))

# Ejemplo con bash
find ~/Downloads -iname "*python*.pdf"

Búsqueda semántica

Para cuando no recuerdas el nombre, pero sí de qué trataba.

  • Sí usa el modelo de lenguaje + embeddings.
  • Busca por el contenido del archivo, no por su nombre.
  • Permite preguntas como: "ese documento sobre redes neuronales".
Flujo:
1. Usuario pregunta: "documento sobre redes neuronales"
2. FileAI busca embeddings similares en ChromaDB
3. Recupera los archivos candidatos
4. El LM resume cada uno
5. Muestra el más relevante al usuario

Fases del proyecto

Fase 1 Escáner de Downloads

Semanas 1–2

  • Escanear ~/Downloads.
  • Guardar metadatos en SQLite.
  • Calcular hashes para detectar duplicados.
  • Extraer texto de PDFs, DOCX y TXT.
  • Generar embeddings y guardarlos en ChromaDB.

Fase 2 Consultas y análisis

Semanas 3–4

  • Estadísticas de espacio y uso de Downloads.
  • Detectar archivos grandes y duplicados.
  • Búsqueda por nombre o parte del nombre.
  • Búsqueda semántica con lenguaje natural.
  • Consultas al LM sobre archivos indexados.

Fase 3 Acciones inteligentes

Semanas 5–6

  • Sugerir archivos a comprimir o borrar.
  • Organizar archivos de Downloads por tipo, fecha o contenido.
  • Detectar y agrupar duplicados.
  • Papelera segura.

Fase 4 Expansión e interfaz

Semanas 7–8

  • Permitir escanear otras carpetas además de Downloads.
  • Dashboard con estadísticas.
  • Chat con el asistente.
  • Visualización de archivos y carpetas.

Instalación paso a paso

1. Crear entorno virtual

cd /home/juan/Documents/proyectosPersonales/fileAI
python3 -m venv .venv
source .venv/bin/activate

2. Instalar Ollama

curl -fsSL https://ollama.com/install.sh | sh

3. Descargar modelos

ollama pull llama3.2
ollama pull nomic-embed-text

4. Crear requirements.txt

typer
chromadb
pymupdf
python-docx
openpyxl
requests
ollama
watchdog

5. Instalar dependencias

pip install -r requirements.txt

6. Verificar Ollama

ollama run llama3.2
# Escribe "hola" y comprueba que responde

7. Primer script de prueba

Crea test_ollama.py:

import ollama

response = ollama.chat(
    model="llama3.2",
    messages=[{"role": "user", "content": "Hola"}]
)
print(response["message"]["content"])

Ejecuta:

python test_ollama.py

Problemas comunes y soluciones

Problema Solución
Downloads tiene demasiados archivos Empieza por subcarpetas pequeñas o limita el número de archivos
Archivos grandes bloquean la extracción Límite de tamaño (ej. 50 MB) y timeout
El modelo alucina respuestas No le des poder de borrar sin confirmación
Carpetas sensibles expuestas Lista negra por defecto: .ssh, .gnupg, etc.
Embeddings ocupan mucho espacio Indexa primero archivos pequeños y recientes
Ollama no responde Verifica que el servicio está activo: ollama serve
Regla de oro: FileAI nunca debe borrar archivos directamente. Siempre mueve a una carpeta de cuarentena y espera confirmación del usuario.
